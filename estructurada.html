Fundamentos de la Programación Estructurada

La programación estructurada es una forma de escribir programas de computadora que comenzó a ser muy popular a partir de 1995. Su objetivo es que el código que se escribe sea fácil de entender y seguir. Es como dar instrucciones claras para hacer una tarea paso a paso. Cuando un programador lee el código, puede comprender cómo funciona el programa, lo que facilita su trabajo. La programación estructurada nació para hacer más fácil la vida de los programadores. Antes, cuando alguien creaba un programa y necesitaba mejorarlo o arreglarlo, podía ser un gran problema porque el código era un enredo difícil de entender. En 1966, dos expertos llamados Böhm y Jacopini descubrieron que cualquier programa se puede hacer usando solo tres tipos de instrucciones. Esto fue increíble porque permitió escribir programas más simples y fáciles de probar. Luego, en 1968, un científico llamado Edsger Dijkstra escribió un artículo muy famoso en el que dijo que usar la instrucción "Goto" era una mala idea. Él ayudó a crear un lenguaje llamado Algol 60, que organizaba mejor los programas. Gracias a su trabajo, los lenguajes de programación se volvieron más ordenados y fáciles de entender. Hoy en día, la programación estructurada es muy usada, al igual que la programación orientada a objetos. Se utiliza en lenguajes muy conocidos como Java, C, Python y C++, que son los que permiten crear programas, juegos y aplicaciones que usamos todos los días.

Variables

Las variables son espacios en la memoria de la computadora donde se guardan datos. Dependiendo del lenguaje de programación, las variables pueden tener diferentes tipos y formas de uso.

Existen dos formas principales de definir variables:

- Tipado estático: El tipo de variable se define al momento de declararla y no puede cambiar.
  • - Tipado dinámico: El tipo de variable se asigna automáticamente cuando el programa se ejecuta.

    • Los tipos más comunes de variables son:

    - Numéricas: Guardan números enteros como 5, 100 o -3.
  • - De punto flotante: Almacenan números con decimales, como 3.14 o -0.5.
    • - De caracteres: Representan símbolos o letras, como "A" o "!".
  • - De cadena de caracteres: Contienen palabras o frases, como "Hola, mundo".
    • - Booleanas: Solo pueden almacenar verdadero o falso.

    Estructuras de control

    Las estructuras de control permiten decidir qué instrucciones ejecutar y cuántas veces hacerlo.

    1. Condiciones: if-else

    Es la estructura más básica para tomar decisiones en un programa.
  • if:
    • # Instrucciones si la condición es verdadera
  • else:
    • # Instrucciones si la condición es falsa

    2. Bucles: while

    Se usa para repetir una acción mientras una condición sea verdadera. cantidad = while cantidad 10: print(cantidad) cantidad += 1 Este código empieza con la variable cantidad en 1. Mientras cantidad sea menor que 10, se muestra en pantalla y se le suma 1. El resultado en pantalla sería: `1 2 3 4 5 6 7 8 9`

    3. Bucles: for

    Permite repetir una acción un número determinado de veces. for cantidad in range(1, 10): print(cantidad) Este código hace lo mismo que el while, pero en una forma más sencilla.

    Funciones

    Las funciones son bloques de código que realizan una tarea específica y pueden reutilizarse muchas veces.

    Características:

  • - Encapsulación: Agrupan instrucciones en una sola unidad.
    • - Reutilización: Se pueden llamar varias veces sin escribir el mismo código.
  • - Legibilidad: Hacen que el programa sea más fácil de entender.
    • - Mantenimiento: Facilitan la detección y corrección de errores. Ejemplo de función: def sumar(a, b): return a + b resultado = sumar(5, 3) print("La suma es:", resultado) Salida: `La suma es: 8`

    Modularidad

    La modularidad consiste en dividir un programa en partes más pequeñas llamadas módulos.

    Ventajas:

  • - Facilita la depuración: Si hay un error, se corrige sin afectar todo el código.
    • - Promueve la reutilización: Un módulo se puede usar en diferentes programas.
  • - Mejor organización: Hace que el código sea más fácil de entender y mejorar.
    • Ejemplo de modularidad:
  • def sumar(a, b): return a + b def restar(a, b): return a - b Luego, en otro archivo se pueden importar y usar estas funciones: import operaciones resultado = operaciones.sumar(10, 5) print("Suma:", resultado) resultado = operaciones.restar(10, 5) print("Resta:", resultado) Salida: `Suma: 15` `Resta: 5`
